# 寻找数组的中心索引

leetcode 原题：1991.find the middle index in array在新窗口打开

题目描述

给你一个下标从 0 开始的整数数组 nums ，请你找到 最左边 的中间位置 middleIndex （也就是所有可能中间位置下标最小的一个）。

中间位置 middleIndex 是满足 nums[0] + nums[1] + ... + nums[middleIndex-1] == nums[middleIndex+1] + nums[middleIndex+2] + ... + nums[nums.length-1] 的数组下标。

如果 middleIndex == 0 ，左边部分的和定义为 0 。类似的，如果 middleIndex == nums.length - 1 ，右边部分的和定义为 0 。

请你返回满足上述条件 最左边 的 middleIndex ，如果不存在这样的中间位置，请你返回 -1 。

示例 1：

输入：nums = [2,3,-1,8,4]
输出：3
解释：
下标 3 之前的数字和为：2 + 3 + -1 = 4
下标 3 之后的数字和为：4 = 4

这题最暴力的解就是两个循环，算出第一个循环语句的下标元素前后元素的和是否相等

export function findMiddleIndex(nums: number[]): number {
  for (let i = 0; i < nums.length; i++) {
    let sum = 0
    for (let j = 0; j < nums.length; j++) {
      if (j < i) {
        sum += nums[j]
      } else if (j > i) {
        sum -= nums[j]
      }
    }
    if (sum === 0) {
      return i
    }
  }
  return -1
}

这种算法的实现时间复杂度为 O(n2)

其实在知道了数组元素总和的情况下，一次遍历这能算出来了。 满足前 n-1 项和的二倍等于总和减去第 n 项的差，则 n 为中心索引

export function findMiddleIndexBetter(nums: number[]): number {
  const total = nums.reduce((acc, cur) => acc + cur, 0)
  let sum = 0
  for (let i = 0; i < nums.length; i++) {
    if (sum * 2 + nums[i] === total) {
      return i
    }
    sum += nums[i]
  }
  return -1
}
